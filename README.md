# Application mobile - AIRNEIS

Ceci est le README générale de l'application mobile

Afin de tester l'application mobile, il vous suffit de récupérer l'ensemble du dossier AIRNEIS-Android qui contient l'entiéreté du code de l'application. Puis il suffit de l'ouvrir avec l'aide d'Android Studio. 

# Installation

Pour lancer le projet, il faut cliquer sur RUN cependant vous avez deux possibilités, soit vous pouvez utiliser l'emulateur android qui est disponible sur Android Studio ou bien vous pouvez lancer le projet sur votre téléphone Android pour cela, il faut activer le mode dévellopeur de celui-ci et brancher votre téléphone en USB. Pour une question d'optimisation et de fluidité, je conseil fortement de faire l'exécution du projet sur un appareil mobile.
Bien vérifier que Android Studio a bien fini la synchronisation des gradle.

Ce dépôt contient le code source et les ressources nécessaires pour le projet Android AIRNEIS. Ce projet vise à la création d'une application de vente e-commerce. Il a été développé en utilisant le langage Java, nous avons utilisé le, et suit les bonnes pratiques de développement Android.

# Fonctionnalités

Les fonctionnalités de cette application sont la consultation de catégorie, de produit. L'insciption d'un compte client ainsi que la connexion, la consultation du compte client est également possible. L'une des fonctionnalités principales d'un site e-commerce est de faire de la vente donc il est possible d'ajouter des produits dans le panier et de passer une commande fictive car il n'y a pas de module de paiement intégré.

# Configuration de l'application
Quand l'utilisateur n'est pas connecté à son compte client il ne peut que consulter les catégories, les articles, les CGV, mentions légales et accéder à la page de contact. 
Une fois qu'il est connecté à un compte il peut ajouter un produit au panier, procéder au paiement, consulter son compte client avec les informations personnelles tel que les moyens de paiement ou les adresses de livraison et l'historique des commandes. 

# Structure du projet

Le projet est constitué de répertoires standart pour une application Android : 

**app**: Contient le code source Spécifique à l'application Android.

**gradle**: Contient les fichiers de configuration Gradle.

Le dossier app contient plusieurs sous dossier qui ont bessoin d'être décrit afin de me comprendre la disposition des fichiers dans le code.

**java** contient toutes les classes et interfaces qui permettent d'executer l'ensemble du projet.

**features**: présent dans le sous dossier java, il regroupe toutes les classes et interface des différentes listes du projet, comme la liste des catégories ou des produits par exemple. 

**modeles**: présent dans le sous dossier java, il regroupe l'ensemble des classes qui se rapporte à la BDD(base de données).

**res**: qui rassemble l'ensemble des éléments graphiques

**drawable**: ensemble des icones

**layout**: ensemble des layout de fragment

**menu**: il s'agit layout de menu burger

**values**: pour les string en Anglais et la traduction française.
